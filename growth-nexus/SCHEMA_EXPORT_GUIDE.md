# 🗄️ دليل سحب هيكل قواعد بيانات Supabase / PostgreSQL

هذا الدليل يشرح كيفية سحب واستخراج "هيكل" قاعدة البيانات (الجداول، العلاقات، الأنواع Enums، الدوال Functions، وصلاحيات RLS) في ملف SQL واحد للتمكن من إعادة بناء المشروع بسهولة.

## 📁 الملفات المطلوبة

لقد قمنا بكتابة ملفين يعملان ببرمجة Node.js، ويمكنك نسخ أي منهما واستخدامه في أي مشروع خارجي يعتمد على قواعد بيانات PostgreSQL أو Supabase.

1. **`generate_schema.js`** (الخيار الأفضل والموصى به):
   - يعتمد فقط على حزمة `pg`.
   - يقرأ من الـ Database ويولد ملف الـ `SQL` برمجياً بالكامل.
   - لا يحتاج إلى تثبيت أي أدوات خارجية مثل `pg_dump` أو `supabase cli`.

2. **`dump_schema.js`** (الخيار البديل):
   - عبارة عن غلاف (Wrapper) يستخدم أداة `pg_dump`.
   - يعمل بشكل ممتاز وسريع، لكنه **يشترط** أن تكون أداة PostgreSQL/pg_dump مثبتة على جهازك ومُعرفة في بيئة النظام (Environment Variables).

## 🚀 كيفية الاستخدام الخارجي (لأي مشروع آخر)

لتشغيل السكربت على قاعدة بيانات أي مشروع، اتبع خطوات التجهيز التالية:

### الخطوة 1: انسخ الملف إلى مشروعك
خذ نسخة من ملف `generate_schema.js` وضعه في المجلد الرئيسي لأي مشروع تريده.

### الخطوة 2: تثبيت الحزم المطلوبة
افتح الشاشة السوداء (Terminal/CMD) في مسار المشروع الجديد ونفذ الأمر التالي لتثبيت مكتبة `pg`:

```bash
npm install pg
```

### الخطوة 3: التنفيذ لسحب الهيكل
استخدم الأمر التالي في الـ Terminal لتشغيل السكربت. 
يجب عليك استبدال `"postgres://..."` برابط الاتصال الخاص بقاعدة بيانات المشروع (Connection URL)، ويمكنك تحديد اسم الملف الناتج (مثلاً `my_database_schema.sql`).

```bash
node generate_schema.js "postgres://user:password@host:port/dbname" "my_database_schema.sql"
```

**مثال تطبيقي للـ Supabase:**
```bash
node generate_schema.js "postgresql://postgres.xxx:YOUR_PASSWORD@aws-0-eu-central-1.pooler.supabase.com:6543/postgres" "export.sql"
```

## 🎯 النتيجة المتوقعة

بعد ثوانٍ قليلة، سيتم إنشاء ملف `export.sql` في مسار المشروع، وسيحتوي على الآتي بشكل منظم:
1. أنواع Enums المخصصة.
2. جداول المشروع مع أنواع الأعمدة والقيم الافتراضية.
3. قيود المفاتيح الأساسية (Primary Keys) والمفاتيح الأجنبية (Foreign Keys) والقيود الفريدة (Unique Constraints).
4. تفعيل سياسات الأمان على مستوى الصف (Row Level Security - RLS).
5. الدوال والـ Triggers (Functions & Triggers) الخاصة بقاعدة البيانات.

بإمكانك أخذ هذا الملف مباشرة ووضعه في أي مشروع Supabase آخر لنسخ نفس بنية قاعدة البيانات بالكامل.

### ملاحظات هامة:
* هذا السكربت يقوم بسحب **"الهيكل والتصميم" (Schema)** ولا يسحب **"البيانات" (Data)** التي بداخل الجداول.
* إذا أردت سحب البيانات مع الهيكل، أو كنت تفضل استخدام أداة قواعد البيانات الرسمية، يرجى تثبيت أداة `pg_dump` واستخدام الأمر: `pg_dump "postgres://..." -O -x -f "dump.sql"`
